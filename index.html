学习渗透测试的第二天 -- SQL注入攻击

Posted on 2025-07-26 Edited on 2025-07-31

**什么是SQL注入攻击：**开发者在开发过程中直接将url的get参数和post参数（用户输入的）直接与SQL语句进行拼接

SQL注入分类：

1.可回显注入：联合查询注入、报错注入、通过注入进行dns请求，达到可回显目的

2.不可回显的注入：bool盲注、时间盲注

3.二次注入：自己编写脚本实现自动化注入

这里书中提到了waf（wab应用防火墙 – 是），觉得要了解一下基本的机制：*WAF基本原理与部署方式-CSDN博客

就这篇文章进行学习：（主要记录不好理解的部分和以前没听过的部分，所以不成系统，概述性的了解，并未对其中的连接性介绍进行拆解）

​ 1。功能概述：流量过滤、安全审计、防止cc攻击（攻击者大量请求->程序消耗大->服务器拒绝服务Doss）、应用交付网络（优化网络、加速设备）

​ 2.作用：保障业务可靠性、可用性、安全性

​ 3.主要功能：常见10大漏洞的防护 发现攻击IP锁定 防止CC攻击

​ 4.发展历程：（IPS架构、反向代理、透明代理、流模式）目前国内的透明代理模式并不如国外的流模式waf，缺失主要在于会改变数据包

联合查询sql注入–以例说明（dvwa、SQL injection、low）

1.注入点判断，数字型/字符型，正常来讲带闭合符号的判断，数字型判断正误正确则为数字型，反之则为字符型。

  • 正常输入判断正确判断条件，返回正常
  • 输入错误的片段条件，返回值仍然正常
  • 说明这个判断为字符型，不是数字型

2.判断字段数（原因： union有严格的约束条件，必须保证字段数一致）

  • 可以使用**oerder by number （例：1’ order by 1#）**这种类型的语句，逐层递加number直到报错，报错前的number即为字段数。
  • 具体内容如下：
  • 正常回显，无报错
  • 正常回显，无报错
  • 应报错，具体报错如下图：说明此字段数应有两位。

3.查看闭合类型：（原理：输入闭合符号进行测试，只要数字后边的字符不是闭合符的，数据库都会把错误的数据转化成正确的数据类型，例1 and 1=2#，返回值是正确的，但是若输入的在数字后面的字符恰好是闭合符，则会形成闭合，若闭合后形成的sql语句是错误的，那么sql语句执行就会错误，从而造成页面显示错误）大部分闭合都是单引号

  • 输入以下图片中的语句，则会报错，因为数据库的查询语句处理不来**1=2#**这个错误语句

4.查看回显位（查看正常有几个可以正常返回查询数据的点位，要求前边的查询语句是非法的，所以只执行后边的语句）

  • 查询语句如下，-1’ union select 1,2#，具体如下图所示：
  • 回显正常、
  • 当输入 **-1’ union select 1,2,3#**回显异常，因为超出了可回显的范围

5.查数据库库名()

  • 输入语句，因为判断有两个回显位，1占一个，另一个函数输出所有的数据库名**-1’ union select 1,group_concat(database())#**记得加注释符，没有注释符会报错。
  • 回显如下，返回数据库名”dvwa”
  • 可以使用group_concat()将所有内容写入一行并输出

  • information_schema是mysql自带的库，里边存放着所有的表名和字段名（列名）
